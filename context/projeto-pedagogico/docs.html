Python Fundamentos da linguagem
f-strings e format specs interpolação e formatação de strings

F-strings embutem expressões Python dentro de strings com {}. O que vem depois dos : é o format spec — controla como o valor é renderizado.

f"texto {expressão:format_spec}"
06d
Inteiro com largura 6, preenchido com zeros. 1 → "000001"
.2f
Float com 2 casas decimais. 3.14159 → "3.14"
.2%
Percentual com 2 casas. 0.067 → "6.70%"
,d
Inteiro com separador de milhar. 1500000 → "1,500,000"
!r
Usa repr() — bom para debug, mostra aspas em strings.
Exemplos
# Interpolação básica
name = "Vini"
f"Olá, {name}"                        # → "Olá, Vini"

# IDs com zero-padding
n = 1
f"APP-{n:06d}"                        # → "APP-000001"
f"APP-{1000:06d}"                     # → "APP-001000"

# Números financeiros
taxa = 0.067
f"{taxa:.2%}"                         # → "6.70%"
f"{taxa:.0%}"                         # → "7%"

renda = 1234567.89
f"R$ {renda:,.2f}"                    # → "R$ 1,234,567.89"

# Expressões dentro das chaves
itens = [1, 2, 3]
f"Total: {len(itens)}"                # → "Total: 3"
f"Dobro: {n * 2}"                     # → "Dobro: 2"

# Debug (Python 3.8+)
valor = 42
f"{valor=}"                           # → "valor=42"
List comprehensions · generator expressions criar sequências de forma declarativa
[expr for item in iterável if condição] → list (expr for item in iterável if condição) → generator
[...] lista
Cria e armazena todos os valores em memória. Use quando precisar indexar, medir com len(), ou iterar várias vezes.
(...) generator
Lazy — produz um valor por vez, sem guardar em memória. Use dentro de all(), any(), sum(), max().
all(v < 5 for v in valores) para de avaliar no primeiro False (short-circuit). Com lista, avaliaria todos antes de retornar.
Exemplos
# Lista básica
quadrados = [x**2 for x in range(5)]          # [0, 1, 4, 9, 16]

# Com filtro
pares = [x for x in range(10) if x % 2 == 0]  # [0, 2, 4, 6, 8]

# Transformação de strings
nomes = [n.strip().lower() for n in raw_names]

# Generator em all() — curto-circuito no primeiro False
all(v < 5.0 for v in [1.2, 3.4, 4.9])         # True
all(v < 5.0 for v in [1.2, 3.4, 6.1])         # False (para em 6.1)

# Generator em sum()
total = sum(len(linha) for linha in linhas)

# Alternativa: dict comprehension
null_dict = {col: pct for col, pct in null_pct.items() if pct > 0}
with — context managers garantir abertura e fechamento de recursos

O bloco with garante que o recurso seja liberado ao sair, mesmo em caso de exceção. Equivalente a try/finally, mas mais legível.

with gerenciador as nome:
open(path, mode)
Modos: 'r' leitura (padrão), 'w' escrita (cria/sobrescreve), 'a' append, 'rb'/'wb' binário.
'w' sobrescreve sem aviso se o arquivo já existir. Para não perder dados, use 'a' (append) ou verifique com Path.exists() antes.
Exemplos
# Escrever JSON em arquivo
with open("saida.json", "w", encoding="utf-8") as f:
    json.dump(records, f, ensure_ascii=False)

# Ler arquivo (modo 'r' é padrão)
with open("data.json") as f:
    data = json.load(f)

# Append — não sobrescreve
with open("log.jsonl", "a") as f:
    f.write(json.dumps(registro) + "\n")

# Dois recursos no mesmo with
with open("entrada.txt") as fin, open("saida.txt", "w") as fout:
    fout.write(fin.read().upper())

Operadores booleanos — Python vs pandas por que and/or falham em Series

and / or avaliam um objeto inteiro como verdadeiro ou falso. Em pandas, a operação precisa acontecer elemento a elemento — por isso use os operadores bitwise.

& (E)
Equivalente ao and para Series. cond1 & cond2
| (OU)
Equivalente ao or. cond1 | cond2
~ (NÃO)
Equivalente ao not. ~cond
Sempre coloque cada condição em parênteses. Sem parênteses, a precedência dos operadores bitwise gera resultados silenciosamente errados.
Exemplos
# ERRADO — raises ValueError: truth value of Series is ambiguous
df[df['age'] > 18 and df['income'] > 0]

# CORRETO — cada condição entre parênteses
df[(df['age'] > 18) & (df['income'] > 0)]          # E
df[(df['age'] < 18) | (df['age'] > 100)]           # OU
df[~df['name'].isnull()]                            # NÃO (negação)

# Múltiplas condições — legível com variável intermediária
mask = (
    (df['age'] >= 18) &
    (df['income'] > 0) &
    (df['defaulted'] == 0)
)
df[mask]

# .sum() em Series booleana conta os True
(df['age'] <= 0).sum()                             # qtd de idades inválidas
bad = ((df['age'] < 18) | (df['age'] > 100)).sum()

# Python puro (fora de pandas) — and/or funcionam normalmente
if x > 0 and y < 100:
    print("ok")

pd.read_csv() carrega CSV em DataFrame
pd.read_csv(filepath, index_col=None, dtype=None, parse_dates=False, na_values=None) → DataFrame
index_col
0 usa a primeira coluna como índice, eliminando a coluna Unnamed: 0 que o Kaggle inclui. False força índice numérico mesmo se houver uma coluna explícita.
dtype
Dict coluna → tipo. Evita inferência que pode ser lenta ou errada. {'id': str, 'score': float}
na_values
Strings adicionais a tratar como NaN. Padrão já inclui '', 'NA', 'null'.

df = pd.read_csv('data.csv', index_col=0)

df = pd.read_csv('data.csv',
    dtype={'applicant_id': str, 'score': float},
    parse_dates=['created_at'])

# Valores nulos customizados
df = pd.read_csv('data.csv', na_values=['', 'N/A', '-', 'none', 'NULL'])

# Ler só algumas colunas (mais rápido em arquivos grandes)
df = pd.read_csv('data.csv', usecols=['id', 'income', 'defaulted'])

# Separador diferente (TSV, ponto-e-vírgula)
df = pd.read_csv('data.tsv', sep='\t')
df = pd.read_csv('data.csv', sep=';')         # padrão europeu

df.isnull() · df.isna() · df.notnull() detectar valores ausentes
DataFrame.isnull() → DataFrame[bool]

isnull() e isna() são aliases exatos. Retornam DataFrame/Series do mesmo shape com True onde o valor é NaN, None, ou NaT.

Para calcular percentual: divida por len(df), não por .count(). len(df) é o total de linhas; .count() já exclui os nulos — você mediria nulo em relação ao não-nulo.
Exemplos
# Booleanos por elemento
df['income'].isnull()               # Series de True/False

# Contar nulos por coluna
df.isnull().sum()

# Percentual de nulos por coluna
(df.isnull().sum() / len(df) * 100).round(2)

# Filtrar linhas onde income é nulo
df[df['income'].isnull()]

# Filtrar linhas onde income NÃO é nulo
df[df['income'].notnull()]
df[~df['income'].isnull()]          # equivalente com negação

# Colunas que TÊM pelo menos 1 nulo
df.columns[df.isnull().any()]

# Filtrado como dict (só colunas com nulos)
null_pct = (df.isnull().sum() / len(df) * 100).round(2)
null_pct[null_pct > 0].to_dict()   # {'MonthlyIncome': 19.82, ...}
Series.fillna() substituir valores ausentes
Series.fillna(value=None, method=None, inplace=False) → Series
value
Escalar, dict ou Series. Escalar preenche todos os NaN com o mesmo valor. Dict mapeia índice → valor.
method
'ffill': propaga o último valor válido para frente. 'bfill': propaga o próximo valor válido para trás. Para séries temporais.
inplace
Se True, modifica o objeto e retorna None. Prefira evitar — retornar e atribuir é mais explícito.
fillna() retorna uma nova Series. Para que a mudança persista no DataFrame, atribua de volta: df['col'] = df['col'].fillna(valor).
Exemplos
# Escalar fixo
df['dependents'] = df['dependents'].fillna(0)

# Mediana da própria coluna (robusto a outliers)
df['income'] = df['income'].fillna(df['income'].median())

# Média
df['score'] = df['score'].fillna(df['score'].mean())

# Valor mais frequente (moda)
df['tier'] = df['tier'].fillna(df['tier'].mode()[0])

# Preencher múltiplas colunas de uma vez
df = df.fillna({
    'income': df['income'].median(),
    'dependents': 0,
    'score': 500
})

# Forward-fill (para séries temporais ordenadas)
df['preco'] = df['preco'].fillna(method='ffill')

# ERRADO — sem atribuição, o DataFrame não muda
df['income'].fillna(0)              # resultado descartado
df['income']                        # ainda tem NaN
Series.astype() converter tipo de dados
Series.astype(dtype) → Series
dtype
Tipo Python (int, float, str, bool) ou string numpy ('int32', 'float64'). 'Int64' (maiúsculo) é o tipo inteiro nullable — suporta NaN.
Converter para int com NaN presentes levanta ValueError. Preencha com .fillna() antes, ou use 'Int64' (nullable integer).
Exemplos
# bool → int (feature de ausência para ML)
df['income_missing'] = df['income'].isnull().astype(int)
# True → 1, False → 0

# str → int (quando você sabe que é seguro)
df['year'] = df['year_str'].astype(int)

# int → str (para joins, IDs)
df['code'] = df['id'].astype(str)

# float com NaN → int (preenche antes)
df['count'] = df['count'].fillna(0).astype(int)

# float → nullable int (mantém NaN)
df['count'] = df['count'].astype('Int64')

# Categoria (economiza memória em colunas repetitivas)
df['tier'] = df['tier'].astype('category')
Series.mean() · median() · sum() · quantile() estatísticas de redução
.mean()
Média aritmética. Ignora NaN por padrão. Em colunas binárias (0/1), a média é diretamente a proporção de 1s.
.median()
Valor que divide a distribuição ao meio. Robusto a outliers — não é afetado por extremos. Para dados financeiros assimétricos, é melhor que a média como valor de imputação.
.sum()
Soma dos valores. Em Series booleanas, conta os True.
.quantile(q)
Valor no percentil q. Float (0.5 = mediana) ou lista de floats. Retorna escalar ou Series.
Exemplos
# Taxa de default — coluna binária: mean() = proporção
df['defaulted'].mean()                             # 0.0671 = 6.71%

# Mediana é robusta a salários extremos
df['income'].median()                              # valor típico
df['income'].mean()                                # puxado para cima por outliers

# Contar com sum() em booleanos
(df['age'] <= 0).sum()                             # idades inválidas
(df['income'] < 0).sum()                           # rendas negativas

# Percentil único
df['income'].quantile(0.99)                        # P99

# Múltiplos percentis de uma vez
df['income'].quantile([0.25, 0.5, 0.75, 0.99])
# 0.25     3000.0
# 0.50     5400.0
# 0.75     8500.0
# 0.99    45000.0

# Por grupo
df.groupby('tier')['income'].median()

# Ignorar nulos (padrão) vs incluir
df['income'].mean(skipna=True)   # padrão
df['income'].mean(skipna=False)  # NaN se tiver qualquer nulo
Series.value_counts() frequência de cada valor único
Series.value_counts(normalize=False, dropna=True) → Series
normalize
Se True, retorna proporções em vez de contagens.
dropna
Se False, NaN aparece como uma categoria no resultado.

df.groupby() agregar por categoria
DataFrame.groupby(by, as_index=True) → GroupBy

groupby() retorna um objeto GroupBy — não um resultado ainda. Aplique uma função de agregação depois.

Exemplos
# Agregação simples
df.groupby('tier')['income'].median()

# Múltiplas colunas, uma função
df.groupby('tier')[['income', 'score']].mean()

# Funções diferentes por coluna
df.groupby('tier').agg({
    'income':    'median',

    'id':        'count'     # volume por tier

# Múltiplas funções numa coluna — resultado tem MultiIndex
df.groupby('tier')['income'].agg(['mean', 'median', 'std'])

# Reset para DataFrame flat (remove MultiIndex do índice)
df.groupby('tier')['income'].mean().reset_index()

# Filtrar grupos com transform (mantém shape original)
df['tier_median'] = df.groupby('tier')['income'].transform('median')
Series.to_dict() · df.to_dict() converter para dicionário Python nativo
Series.to_dict() → dict DataFrame.to_dict(orient) → dict | list[dict]
Use to_dict() quando precisar serializar para JSON ou passar para código que não depende de pandas. Series e DataFrames não são serializable por padrão.
Exemplos
# Series → {índice: valor}
null_pct.to_dict()
# {'MonthlyIncome': 19.82, 'NumberOfDependents': 3.11}

# Padrão comum: filtrar antes de converter
null_pct[null_pct > 0].to_dict()   # só as colunas com nulos

# DataFrame → list de dicts (uma por linha) — o mais comum para APIs
df.head(3).to_dict('records')
# [{'id': 1, 'income': 5000, ...}, {'id': 2, ...}, ...]

# DataFrame → dict de dicts (coluna → {índice → valor})
df.to_dict('dict')

# Agregar e converter diretamente
df.groupby('tier')['income'].median().to_dict()
# {'HIGH': 7200.0, 'LOW': 3100.0, 'MEDIUM': 5000.0}
df.to_parquet() · pd.read_parquet() persistência colunar eficiente
DataFrame.to_parquet(path, index=True, compression='snappy') pd.read_parquet(path, columns=None) → DataFrame
index=False
Não salva o índice numérico sequencial. Use sempre que o índice não tiver significado semântico.
columns
Lista de colunas a ler. Parquet é colunar — você paga só pelo que lê. Muito mais rápido que CSV em tabelas largas.
Requer pip install pyarrow.
Exemplos
# Salvar sem o índice numérico
df.to_parquet('output.parquet', index=False)

# Com compressão mais forte (arquivo menor, mais lento para ler/escrever)
df.to_parquet('output.parquet', index=False, compression='gzip')

# Sem compressão (mais rápido para I/O, arquivo maior)
df.to_parquet('output.parquet', index=False, compression=None)

# Ler tudo
df = pd.read_parquet('output.parquet')

# Ler só colunas necessárias (muito mais rápido em arquivos grandes)
df = pd.read_parquet('output.parquet', columns=['id', 'income', 'defaulted'])

df.merge() juntar dois DataFrames por coluna-chave
DataFrame.merge(right, how='inner', on=None, left_on=None, right_on=None, suffixes=('_x','_y')) → DataFrame
how
'inner': só linhas que casam nos dois. 'left': todas da esquerda, NaN para não-casados. 'outer': tudo de ambos.
on
Nome da coluna presente nos dois. Use left_on / right_on quando os nomes diferem.
suffixes
Sufixos para colunas com nome duplicado fora da chave. Padrão ('_x', '_y'); personalize para clareza.
Colunas com nome igual que não são chave ganham sufixo automático. Use suffixes=('_csv', '_api') para clareza em vez de _x/_y.
Exemplos
# Inner join — só linhas com match nos dois
df = df_csv.merge(df_api, on='applicant_id')

# Left join — mantém todos os registros do CSV (NaN onde não há match)
df = df_csv.merge(df_api, on='applicant_id', how='left')

# Chaves com nomes diferentes
df = df_csv.merge(df_api, left_on='applicant_id', right_on='id')

# Sufixos legíveis (evita income_x, income_y)
df = df_csv.merge(df_api, on='id', suffixes=('_csv', '_api'))

# Checar se o join não explodiu o número de linhas
print(f"csv={len(df_csv)}, api={len(df_api)}, merged={len(df)}")
# Se merged > csv → havia duplicatas na chave do lado direito

# Merge em múltiplas colunas
df = df1.merge(df2, on=['applicant_id', 'data_ref'])

# Outer join — todas as linhas de ambos (NaN onde não casar)
df = df_csv.merge(df_api, on='id', how='outer', indicator=True)
# Coluna '_merge': 'left_only', 'right_only', 'both'
df[df['_merge'] == 'right_only']   # registros que estão só na API
df.loc[] · df.iloc[] · df.at[] selecionar por rótulo ou por posição inteira
df.loc[row_label, col_label] df.iloc[row_int, col_int]
.loc[]
Seleção por rótulo (nome do índice ou da coluna). Slices incluem os dois extremos.
.iloc[]
Seleção por posição inteira (0-based). Slices excluem o limite superior — igual ao Python puro.
.at[] / .iat[]
Versões escalares de .loc / .iloc. Mais rápidas quando se quer um único valor.
Quando o índice é inteiro sequencial, .loc[0] e .iloc[0] retornam o mesmo. Diferem quando o índice tem gaps, strings, ou após um .reset_index().
Exemplos
# .loc — por rótulo
df.loc[0]                        # linha com label 0
df.loc[0, 'income']              # linha 0, coluna 'income'  → escalar
df.loc[:, 'income']              # coluna inteira
df.loc[df['age'] > 30]           # filtro booleano (forma mais comum)
df.loc[df['tier'] == 'HIGH', 'income']  # filtro + coluna específica

# .iloc — por posição
df.iloc[0]                       # primeira linha
df.iloc[-1]                      # última linha
df.iloc[0, 2]                    # linha 0, coluna índice 2
df.iloc[:5]                      # primeiras 5 linhas
df.iloc[:, :3]                   # primeiras 3 colunas
df.iloc[::2]                     # linhas pares (step=2)

# .at / .iat — escalar direto (mais rápido que loc/iloc)
df.at[0, 'income']               # label 0, coluna 'income'
df.iat[0, 3]                     # posição (0, 3)

# Converter linha para dict (comum em APIs)
df.iloc[0].to_dict()             # {'income': 5000, 'age': 35, ...}
df.query() · df.apply() · Series.map() filtros expressivos e transformações por linha
DataFrame.query(expr: str) → DataFrame DataFrame.apply(func, axis=0) → Series Series.map(arg: dict | callable) → Series
query()
Filtra com string de expressão Python. Legível para múltiplas condições. Variáveis externas via @nome.
apply(axis=1)
Aplica função linha a linha. Lento em grandes DataFrames — prefira operações vetorizadas quando possível.
map(dict)
Mapeia cada valor via dict ou função. Mais rápido que apply para transformações de coluna única.
Exemplos
# query() — legível para condições compostas
df.query("age > 30 and income > 5000")
df.query("risk_tier == 'HIGH' or defaulted == 1")
df.query("risk_tier in ['HIGH', 'MEDIUM']")

# variável externa com @
min_age = 18
df.query("age >= @min_age and income > 0")

# apply() por coluna (axis=0, padrão) — preenche nulos com mediana de cada coluna
df[['income', 'score']].apply(lambda col: col.fillna(col.median()))

# apply() por linha (axis=1) — lógica que cruza múltiplas colunas
def classify(row):
    if row['late_90_days'] > 2:         return 'HIGH'
    elif row['late_30_59_days'] > 1:    return 'MEDIUM'
    return 'LOW'
df['risk_tier'] = df.apply(classify, axis=1)

# map() — transformação de uma coluna via dict
label_map = {'HIGH': 2, 'MEDIUM': 1, 'LOW': 0}
df['risk_num'] = df['risk_tier'].map(label_map)

# map() — via função (sem dict)
df['income_k'] = df['income'].map(lambda x: round(x / 1000, 1))

# map() vs apply: map só para Series, apply para DataFrame/Series
pathlib Manipulação de caminhos
Path() objeto de caminho cross-platform
from pathlib import Path
p = Path(string)

Representa caminhos de forma orientada a objetos. O operador / concatena segmentos.

Exemplos
from pathlib import Path

# Criar objeto Path
p = Path('data/raw/file.csv')

# Concatenar com / (funciona em Windows e Unix)
p = Path('data') / 'raw' / 'file.csv'

# Acessar partes
p.name      # 'file.csv'        — nome com extensão
p.stem      # 'file'            — nome sem extensão
p.suffix    # '.csv'            — extensão (com ponto)
p.parent    # Path('data/raw')  — diretório pai

# Verificar existência
if p.exists():
    df = pd.read_csv(p)

p.is_file()   # True se é arquivo
p.is_dir()    # True se é diretório

# Listar arquivos num diretório
list(Path('data/raw').glob('*.csv'))      # todos os .csv
list(Path('data').rglob('*.parquet'))    # recursivo
Path.mkdir() criar diretório
Path.mkdir(parents=False, exist_ok=False)
parents=True
Cria todos os diretórios intermediários, como mkdir -p.
exist_ok=True
Não levanta exceção se o diretório já existir. Torna o código idempotente.
Em produção, sempre use mkdir(parents=True, exist_ok=True) antes de escrever. É idempotente — pode ser chamado quantas vezes quiser.

hashlib Hash criptográfico
hashlib.sha256() hash determinístico e one-way
hashlib.sha256(data: bytes).hexdigest() → str (64 chars)
data
Deve ser bytes. Converta string com .encode() (UTF-8 por padrão).
.hexdigest()
Hash como string hexadecimal de 64 chars. Truncar com [:16] reduz para 16 chars — suficiente como identificador interno.
Determinístico: mesmo input → mesmo hash sempre. One-way: impossível recuperar o dado original. Por isso é o padrão para anonimizar PII mantendo capacidade de join (mesmo CPF → mesmo hash).
Exemplos
import hashlib

# Hash básico de string
hashlib.sha256("hello".encode()).hexdigest()
# → 'b94d27b9934d3e08a52e52d7da7dabfac484efe04294e576...' (64 chars)

# Com truncação para identificador interno
hashlib.sha256("hello".encode()).hexdigest()[:16]
# → 'b94d27b9934d3e08'  (16 chars)

# Anonimizar CPF (PII) mantendo joinability
cpf_raw = "123.456.789-09"
cpf_clean = cpf_raw.replace('.', '').replace('-', '')  # '12345678909'
cpf_hash = hashlib.sha256(cpf_clean.encode()).hexdigest()[:16]

# Verificar que é determinístico
a = hashlib.sha256("texto".encode()).hexdigest()
b = hashlib.sha256("texto".encode()).hexdigest()
a == b   # True — sempre igual para o mesmo input

# Input diferente → hash completamente diferente (avalanche effect)
hashlib.sha256("texto".encode()).hexdigest()[:8]   # '9f86d081'
hashlib.sha256("Texto".encode()).hexdigest()[:8]   # 'b14b957c' — totalmente diferente

# Outros algoritmos disponíveis
hashlib.md5("hello".encode()).hexdigest()           # 32 chars (não usar para segurança)
hashlib.sha512("hello".encode()).hexdigest()        # 128 chars

Window functions · QUALIFY · ROW_NUMBER análise por grupo sem reduzir linhas

Window functions calculam valores usando um grupo de linhas sem colapsá-las. A cláusula OVER (PARTITION BY ... ORDER BY ...) define a janela.

ROW_NUMBER()
Número sequencial dentro da partição. 1, 2, 3… sem empates.
RANK()
Posição com empates — dois primeiros empatados dão RANK=1, próximo é RANK=3.
LAG(col, n)
Valor da linha n posições atrás. Útil para comparações temporais.
QUALIFY
Filtra resultado de window function direto no SELECT — sem subconsulta. Único em DuckDB/Snowflake.
QUALIFY ROW_NUMBER() OVER (...) = 1 é a forma idiomática do DuckDB para deduplicar sem CTE.
Exemplos
-- Numerar linhas por partição
SELECT applicant_id, age, monthly_income,
       ROW_NUMBER() OVER (PARTITION BY risk_tier ORDER BY monthly_income DESC) AS rn
FROM mart_credit_features;

-- QUALIFY: filtrar o resultado da window function direto (sem subquery)
SELECT *
FROM 'data/staging/credit_applications.parquet'
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY age, monthly_income, debt_ratio
    ORDER BY 1
) = 1;
-- equivale a: "pega só a primeira ocorrência de cada combinação" — remove duplicatas

-- Ranking de renda dentro de cada tier
SELECT applicant_id, risk_tier, monthly_income,
       RANK() OVER (PARTITION BY risk_tier ORDER BY monthly_income DESC) AS income_rank
FROM mart_credit_features
QUALIFY income_rank <= 3;    -- top 3 por tier

-- LAG: comparar valor com linha anterior (útil em séries históricas)
SELECT run_id, default_rate_pct,
       LAG(default_rate_pct, 1) OVER (ORDER BY timestamp) AS prev_rate,
       default_rate_pct - LAG(default_rate_pct, 1) OVER (ORDER BY timestamp) AS delta
FROM 'data/monitoring/metrics_history.parquet';

-- SUM acumulado (running total)
SELECT applicant_id, monthly_income,
       SUM(monthly_income) OVER (ORDER BY applicant_id) AS running_total
FROM mart_credit_features;

Prefect Orquestração de pipelines
@task · @flow decoradores de orquestração
@task
Unidade atômica. Prefect monitora estado (Pending→Running→Completed/Failed) e captura logs de cada task individualmente.
@flow
Orquestrador que chama tasks em sequência. Se uma task falhar, o flow para naquele ponto — não silenciosamente.
retries
Tentativas automáticas antes de marcar como Failed.
Tasks passam dados umas para as outras via retorno normal de função. O Prefect rastreia as dependências automaticamente a partir do grafo de chamadas.
Exemplos
from prefect import task, flow
import pandas as pd, subprocess

@task(name="Ingest CSV", retries=2)
def ingest(path: str):
    df = pd.read_csv(path, index_col=0)
    print(f"CSV: {len(df)} registros")
    return df                                  # valor passado para a próxima task

@task(name="Validate")
def validate(df):
    from src.ingestion import validate_dataframe
    report = validate_dataframe(df)
    print(f"Qualidade: {report}")
    return report

@task(name="Run Tests")
def run_tests():
    result = subprocess.run(['pytest', 'tests/', '-v'],
                            capture_output=True, text=True)
    print(result.stdout)
    if result.returncode != 0:
        raise Exception("Testes falharam — pipeline abortado")

@flow(name="Credit Pipeline")
def credit_pipeline():
    df      = ingest("data/raw/cs-training.csv")  # executa e aguarda
    report  = validate(df)                         # df passado como argumento
    run_tests()                                    # executa por último
    return report

# Executar diretamente
if __name__ == "__main__":
    credit_pipeline()

@pytest.fixture
Função que prepara dados ou recursos para os testes. Injetada automaticamente pelo nome do parâmetro. Com scope='module', executa uma vez por arquivo — economiza tempo em I/O (ler Parquet).
@pytest.mark.parametrize
Roda o mesmo teste com múltiplos inputs. Cada par de valores gera um teste separado no relatório.
Exemplos
import pandas as pd, pytest

# Fixture — carrega Parquet uma vez para todos os testes do módulo
@pytest.fixture(scope='module')
def mart():
    return pd.read_parquet('data/marts/mart_credit_features.parquet')

# Fixture injetada pelo nome do parâmetro
def test_no_nulls(mart):
    assert mart['risk_tier'].isnull().sum() == 0

def test_record_count(mart):
    assert len(mart) > 1000, "Mart tem menos registros do que o esperado"

# Fixture com múltiplos valores — testa cada coluna crítica separada
@pytest.fixture(params=['applicant_id', 'defaulted', 'risk_tier', 'total_late_payments'])
def critical_col(request):
    return request.param

def test_critical_cols_no_nulls(mart, critical_col):
    count = mart[critical_col].isnull().sum()
    assert count == 0, f"{critical_col} tem {count} nulos"

# parametrize — mais conciso para o mesmo padrão
@pytest.mark.parametrize("col", ['applicant_id', 'defaulted', 'risk_tier'])
def test_col_no_nulls(mart, col):
    assert mart[col].isnull().sum() == 0

# parametrize — testar múltiplos pares (input, esperado)
@pytest.mark.parametrize("tier, max_rate", [
    ('LOW',    0.10),
    ('MEDIUM', 0.30),
    ('HIGH',   1.00),
])
def test_default_rate_by_tier(mart, tier, max_rate):
    subset = mart[mart['risk_tier'] == tier]
    rate = subset['defaulted'].mean()
    assert rate <= max_rate, f"Default rate de {tier}: {rate:.2%} > {max_rate:.2%}"
logging Logs estruturados
logging.getLogger() · basicConfig() · níveis logging padrão da stdlib

O módulo logging da stdlib tem 5 níveis: DEBUG (10) → INFO (20) → WARNING (30) → ERROR (40) → CRITICAL (50). Só mensagens acima do nível configurado são emitidas.

getLogger(name)
Retorna o logger com esse nome (hierárquico por ponto: 'pipeline.ingest'). Loggers com o mesmo nome são o mesmo objeto.
basicConfig()
Configura o logger root — afeta todos os módulos. Use apenas em scripts, não em bibliotecas.
Exemplos

)

# Logger por módulo (boa prática)

python-json-logger · run_id · Cloud Logging logs estruturados em JSON para produção
pip install python-json-logger

Logs em texto são legíveis para humanos. Logs em JSON são indexáveis por ferramentas como Cloud Logging, Datadog, Loki. Com um run_id único por execução, você filtra todos os logs de um pipeline específico.

Exemplos
import logging, uuid
from pythonjsonlogger import jsonlogger

def get_logger(name: str, run_id: str = None) -> logging.Logger:
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)
    if not logger.handlers:             # guard contra handlers duplicados
        handler = logging.StreamHandler()
        fmt = jsonlogger.JsonFormatter(
            fmt='%(asctime)s %(name)s %(levelname)s %(message)s',
            rename_fields={
                'asctime':   'timestamp',
                'levelname': 'level',
                'message':   'event'
            }
        )
        handler.setFormatter(fmt)
        logger.addHandler(handler)
    return logger

# Uso em flow.py
RUN_ID = str(uuid.uuid4())[:8]          # ex: 'a3f7b1c2'
log = get_logger("pipeline", RUN_ID)

log.info("CSV ingerido", extra={"run_id": RUN_ID, "records": 150_000})

random Faker Dados aleatórios e mocks
random.randint() · choice() · uniform() · seed() geração controlada de números aleatórios
randint(a, b)
Inteiro aleatório entre a e b inclusive. Ambos os extremos incluídos — diferente do range().
choice(seq)
Elemento aleatório de uma sequência. Levanta IndexError em sequência vazia.
choices(seq, weights, k)
Como choice mas com pesos e quantidade — permite distribuições tendenciosas.
uniform(a, b)
Float aleatório no intervalo [a, b].
seed(n)
Fixa a semente — toda sequência gerada depois é reproducível. Essencial para testes.
Exemplos
import random

# Seed para reproducibilidade (testes, demos)
random.seed(42)

# Inteiros
random.randint(0, 1000)          # bureau_score entre 0 e 1000 (inclusive)
random.randint(18, 80)           # idade aleatória

# Float
random.uniform(0.0, 1.0)        # revolving_utilization entre 0 e 1
random.uniform(0.5, 3.0)        # debt_ratio

# Escolha de lista
random.choice(['LOW', 'MEDIUM', 'HIGH'])

# Escolha com pesos (70% LOW, 20% MEDIUM, 10% HIGH — distribuição realista)
tier = random.choices(
    ['LOW', 'MEDIUM', 'HIGH'],
    weights=[70, 20, 10],
    k=1
)[0]

# Lista de N escolhas com pesos
tiers = random.choices(['LOW', 'MEDIUM', 'HIGH'], weights=[70, 20, 10], k=5000)

# Embaralhar lista in-place
deck = list(range(1000))
random.shuffle(deck)

# Sample sem reposição (subconjunto)
